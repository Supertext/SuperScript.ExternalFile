CacheForTimePeriod option

The CacheForTimePeriod option allows configuration of the Cache-Control header. 

  public TimeSpan CacheForTimePeriod { get; set; }

If set to null then the Cache-Control header will be set to "no-cache".

The default value is 3650 days, Cache-Control: "public".

Configuring CacheForTimePeriod

The CacheForTimePeriod option may be configured programmatically or declaratively in the configuration file.

Configuring CacheForTimePeriod programmatically

The following example shows the creation of a new emitter which is configured with its HtmlWriter property set to an instance of ExternalScriptWriter which has the CacheForTimePeriod property set to 3 days: 

  IEmitter emitter = new StandardEmitter
                         {
                             HtmlWriter = new ExternalScriptWriter
                                              {
                                                  CacheForTimePeriod = new TimeSpan(3, 0, 0, 0)
                                              }
                         }

Declaring CacheForTimePeriod in the config file

The following example shows the same declaration as above, as declared in a config file. 

    <section name="superScript" type="SuperScript.Configuration.SuperScriptConfig, SuperScript.Common" />
    <section name="superScript.ExternalFile" type="SuperScript.ExternalFile.Configuration.ExternalFileConfig, SuperScript.ExternalFile" />
    

    <superScript>
      <emitters>
        <emitter key="externalJavascript">
          <writers>
            <modifier type="SuperScript.ExternalFile.Modifiers.Writers.ExternalScriptWriter, SuperScript.ExternalFile">
              <properties>
                <property name="CacheForTimePeriod" value="3.00:00:00" type="System.TimeSpan, mscorlib" />
              </properties>
            </modifier>
          </writers>
        </emitter>
      </emitters>
    </superScript>

Both the above examples will set the following properties in the HTTP header of the returne file:

Cache-Control:  "public"
Expires:  [a string representation of a date 3 days hence]

When set in the config file, SuperScript parses the string representation of the TimeSpan value using TimeSpan.TryParse(string, out TimeSpan). More can be read here on the options for representing TimeSpan values as string.

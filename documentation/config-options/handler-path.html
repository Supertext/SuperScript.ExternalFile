HandlerPath option

The HandlerPath option allows setting the path of the HTML handler which returns the file. 

  public string HandlerPath { get; set; }

The default value is "/Files/".

The value set for this property must match the value set in the "path" attribute set in the relevand handler as configured in the configuration file.

For example, the configuration file should have the following element: 

  <system.webServer>
    <handlers>
      …
      <add name="GetHandler" verb="GET" path="files" type="SuperScript.ExternalFile.Handlers.Get" />
      …
    </handlers>
  </system.webServer>

If the HandlerPath property is changed from the default value then the "path" attribute as shown in the above snippet must be set to the same value.

Read more about the handler for retrieving the file contents here.

Configuring HandlerPath

The HandlerPath option may be configured programmatically or declaratively in the configuration file.

Configuring HandlerPath programmatically

The following example shows the creation of a new emitter which is configured with its HtmlWriter property set to an instance of ExternalFileWriter which has the HandlerPath property set to "scripts":

It should be remembered that the default file extension is "js". 

  IEmitter emitter = new StandardEmitter
                         {
                             HtmlWriter = new ExternalFileWriter
                                              {
                                                  HandlerPath = "scripts"
                                              }
                         }

Declaring HandlerPath in the config file

The following example shows the same declaration as above, as declared in a config file. 

  <configuration>
    <configSections>
      <section name="superScript" type="SuperScript.Configuration.SuperScriptConfig, SuperScript.Common" />
      <section name="superScript.ExternalFile" type="SuperScript.ExternalFile.Configuration.ExternalFileConfig, SuperScript.ExternalFile" />
    </configSections>
    …
    <system.webServer>
      <handlers>
        …
        <add name="GetHandler" verb="GET" path="scripts" type="SuperScript.ExternalFile.Handlers.Get" />
        …
      </handlers>
    </system.webServer>
    …
    <superScript>
      <emitters>
        <emitter key="externalJavascript">
          <writers>
            <modifier type="SuperScript.ExternalFile.Modifiers.Writers.ExternalFileWriter, SuperScript.ExternalFile">
              <properties>
                <property name="HandlerPath" value="scripts" type="System.String, mscorlib" />
              </properties>
            </modifier>
          </writers>
        </emitter>
      </emitters>
    </superScript>
  </configuration>

Both the above examples will emit the following result on the HTML page at the location where Emit.For("externalJavascript") is called: 

  <script type="/scripts/[hash of the file's contents].js" type="text/javascript"></script>

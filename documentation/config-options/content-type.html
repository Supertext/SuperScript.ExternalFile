ContentType option

The ContentType option allows configuration of the Content-Type MIME header. 

  public string ContentType { get; set; }

The default value is "text/javascript".

Configuring ContentType

The ContentType option may be configured programmatically or declaratively in the configuration file.

Configuring ContentType programmatically

The following example shows the creation of a new emitter which is configured with its HtmlWriter property set to an instance of ExternalFileWriter which has the ContentType property set to "application/xml": 

  IEmitter emitter = new StandardEmitter
                         {
                             HtmlWriter = new ExternalFileWriter
                                              {
                                                  ContentType = "application/xml"
                                              }
                         }

Declaring ContentType in the config file

The following example shows the same declaration as above, as declared in a config file. 

    <section name="superScript" type="SuperScript.Configuration.SuperScriptConfig, SuperScript.Common" />
    <section name="superScript.ExternalFile" type="SuperScript.ExternalFile.Configuration.ExternalFileConfig, SuperScript.ExternalFile" />
    …
    <superScript>
      <emitters>
        <emitter key="externalJavascript">
          <writers>
            <modifier type="SuperScript.ExternalFile.Modifiers.Writers.ExternalFileWriter, SuperScript.ExternalFile">
              <properties>
                <property name="ContentType" value="application/xml" type="System.String, mscorlib" />
              </properties>
            </modifier>
          </writers>
        </emitter>
      </emitters>
    </superScript>

Both the above examples will set the following properties in the HTTP header of the returned file:

Content-Type:  "application/xml"

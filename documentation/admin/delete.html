Administrative API

SuperScript.ExternalFile offers an administrative API for managing the external files.

Delete

SuperScript.ExternalFile allows individual files to be deleted from the content store.

This is achieved by navigating to the following URL: 

  http://{your website URL}/delete/{filename}

If successful then SuperScript.ExternalFile will issue an HTTP 200 (OK) response.

The Delete handler may be re-mapped to a different URL. To do this, simply modify the path attribute of the handler declaration in the config file: 

  <system.webServer>
    <handlers>
      <add name="DeleteHandler" verb="GET" path="delete" type="SuperScript.ExternalFile.Handlers.Delete" />
    </handlers>
  </system.webServer>

Empty

Requests that all contents are deleted from the content store.

This is achieved by navigating to the following URL: 

  http://{your website URL}/empty/

If successful then SuperScript.ExternalFile will issue an HTTP 200 (OK) response.

The Empty handler may be re-mapped to a different URL. To do this, simply modify the path attribute of the handler declaration in the config file: 

  <system.webServer>
    <handlers>
      <add name="EmptyHandler" verb="GET" path="empty" type="SuperScript.ExternalFile.Handlers.Empty" />
    </handlers>
  </system.webServer>

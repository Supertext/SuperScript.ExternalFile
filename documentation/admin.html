Administrative API

SuperScript.ExternalFile offers an administrative API for managing the external files.

Delete a file

SuperScript.ExternalFile allows individual files to be deleted from the content store.

This is achieved by navigating to the following URL: 

  http://{your website URL}/delete/{filename}

If successful then SuperScript.ExternalFile will issue an HTTP 200 (OK) response.

The Delete handler may be re-mapped to a different URL. To do this, simply modify the path attribute of the handler declaration in the config file: 

  <system.webServer>
    <handlers>
      <add name="DeleteHandler" verb="GET" path="delete" type="SuperScript.ExternalFile.Handlers.Delete" />
    </handlers>
  </system.webServer>

Get a file

Gets the specified file from the implementation of IStore.

This is achieved by navigating to the following URL: 

  http://{your website URL}/files/

If successful then SuperScript.ExternalFile will respond with the file, as configured.

The Get handler may be re-mapped to a different URL. To do this, simply modify the path attribute of the handler declaration in the config file: 

  <system.webServer>
    <handlers>
      <add name="GetHandler" verb="GET" path="files" type="SuperScript.ExternalFile.Handlers.Get" />
    </handlers>
  </system.webServer>

  <system.webServer>
    <handlers>

List files in the store

Returns a webpage enumerating all files in the store.

  http://{your website URL}/list/

Re-initialise the store

Causes a request to be made to the IStore implementation to remove and then initialise the store. This request is equivalent to sending a request to Remove followed by a request to Init.

This is achieved by navigating to the following URL: 

  http://{your website URL}/reinit/

If successful then SuperScript.ExternalFile will issue an HTTP 200 (OK) response.

The ReInit handler may be re-mapped to a different URL. To do this, simply modify the path attribute of the handler declaration in the config file: 

  <system.webServer>
    <handlers>
      <add name="ReInitHandler" verb="GET" path="reinit" type="SuperScript.ExternalFile.Handlers.ReInit" />
    </handlers>
  </system.webServer>

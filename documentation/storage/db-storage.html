Database Storage

SuperScript.ExternalFile provides a means for storing the contents of the external file in a database. Such a means is provided by an implementation of IDbStore.

However, although IDbStore offers the methods, properties, etc. for achieving this, an implementation of IDbStore on its own is not sufficient: for storing the contents of external files in a database SuperScript.ExternalFile also requires an object which contains information specific to the type of underlying database, and this is supplied by an implementation of IDbStoreProvider.

First of all, let's take a look at the IDbStore interface which, as is the case with IsoStore and FileStore, derives from IStore. 

	interface IDbStore : SuperScript.ExternalFile.Storage.IStore
	{
		IDbStoreProvider DbStoreProvider { get; set; }
	}

IDbStore adds only a single property to the members provided by IStore, and this is the property which instructs SuperScript.ExternalFile about the underlying database.

Name Type Description
DbStoreProvider IDbStoreProvider

An instance of IDbStoreProvider which contains the instructions for initialising a database-based implementation of IDbStore.

Inherited members
AddOrUpdate void

Adds the specified instance of IStorable to the store using the specified key. If an item exists with the specified Key then it will be updated.

Delete void

Deletes the specified instance of IStorable.

Empty void

Removes all directories and instances of IStorable from the store.

Get IStorable

Gets the instance of IStorable with the specified key.

Returns null if no matching keys were found.

GetAll IEnumerable<IStorable>

Returns a snapshot of all IStorable instances in the store.

Init void

Performs any initilisation that the store requires before items may be stored and retrieved.

Remove void

Removes all traces of the store.

StoreName string

Gets or sets the name of the top-level directory inside the storage provider, should it be required.

Although an implementation of IDbStore is provided out of the box, a configured implementation is not sufficient for storing the contents, and an instance of IDbStoreProvider must also be configured.

Important: SuperScript.ExternalFile does not contain any implementations of IDbStoreProvider.

SuperScript.ExternalFile.SqlServer and SuperScript.ExternalFile.MySql are two examples of IDbStoreProvider.

The DbStore class

SuperScript.ExternalFile provides an implementation of IDbStore, unsurprisingly called DbStore. 

	public class DbStore : SuperScript.ExternalFile.Storage.IDbStore
	{
		AddOrUpdate(IStorable storable);
		IDbStoreProvider DbStoreProvider { get; set; }
		void Delete(string key);
		void Empty();
		IStorable Get(string key);
		IEnumerable<IStorable> GetAll();
		void Init();
		void Remove { get; set; }
		string StoreName { get; set; }
	}

For an explanation of each of these members the IDbStore documentation (above) should be consulted.

The Database Provider

As explained, the IDbStore and the IDbStoreProvider go hand-in-hand. 

	interface IDbStoreProvider
	{
		AddOrUpdate(IStorable storable);
		string ConnectionString { get; set; }
		string DbName { get; set; }
		void Delete(string key);
		void DeleteStore();
		IStorable Get(string key);
		IEnumerable<IStorable> GetAll();
		void Init();
		string StoreName { get; set; }
	}
Name Type Description
AddOrUpdate void

Adds the specified instance of IStorable to the store using the specified key. If an item exists with the specified Key then it will be updated.

ConnectionString string

Gets or sets the connection string that will be used to communicate with the underlying database.

DbName string

Gets or sets the name of the database, if this is not already detailed in the connection string.

Delete void

Deletes the instance of IStorable which has been stored against the specified Key.

DeleteStore void

Deletes the entire store from the database.

Get IStorable

Retrieves an IStorable instance from the database if an instance is found with the matching Key.

GetAll IEnumerable<IStorable>

Returns a snapshot of all IStorable instances in the store.

StoreName string

Gets or sets the name of the table inside the database.

Init void

Checks that the store (a database table) exists. If not, the store will be created.

Example

A practical implementation of IDbStore and IDbStoreProvider can be seen in the SuperScript.ExternalFile.SqlServer project.
